## 1. 기존 철학 (As-Is)

### 1.1 설계 의도

YDS는 9대 패닉·심리 지표를 **0–100 총점**으로 합산하고, 5단계 구간에 매핑해 **역발상 매매 행동**을 제시한다.

### 1.2 기존 메시지 톤 (문제의 소지)

| 구간 | 기존 UI·카피 인상 |
|------|-------------------|
| 🔵 과열 | 비중 축소 |
| 🟢 중립 | 관찰 |
| 🟡 관심 | 관심 종목 |
| 🟠 분할매수 | 매수 구간 |
| 🔴 **패닉매수** | **「강한 매수」「적극 매수」— 클라이맥스** |

→ 사용자·외부 관점: **「YDS = 패닉 오면 사라」** 느낌  
→ 5단계 중 **최종 목표가 패닉매수**처럼 읽힘

### 1.3 엔진 실제 (검증 결과)

| 사실 | 수치 (2019–2026) |
|------|------------------|
| 패닉매수(80+) 체류 | **0%** |
| 분할매수(60–79) 체류 | **12.3%** |
| 관심(40–59) 체류 | **43.9%** |
| COVID VIX 83 → YDS | **75 (분할매수)** |

**→ 엔진은 패닉 탐지기가 아니라, 관심·분할 중심의 사이클 엔진으로 이미 동작 중.**

## 2. 문제점

### 2.1 철학–엔진 불일치

```
[UI/내러티브]  ──→  패닉매수 = 최종 목표
[엔진/역사]    ──→  패닉매수 ≈ never · 분할매수 = 실전 클라이맥스
```

### 2.2 기회 상실

- 사용자가 **80+를 기다리면** 7년간 **0회** 진입
- 실제 고수익 구간(관심 +22%, 분할 +39% / 12M ^GSPC)을 **「아직 패닉 아님」** 으로 스킵

### 2.3 패닉 편향 인식

- VIX 급등·지수 -3%인데 YDS 38(중립) → **「엔진 고장?」** 오해
- 실제: HY·BofA·동적 가중 **의도적 완화** — 극단 공포 누적 전까지 사이클 중·상단 유지

---

## 3. 시장 사이클 엔진 철학 (To-Be)

### 3.1 정의

> **YDS는 패닉 탐지기가 아니라, 과열 → 중립 → 관심 → 분할매수 → (보너스) 패닉매수로 이어지는 시장 사이클 투자 엔진이다.**

- **9개 지표** → **YDS 총점** → **행동 가이드**
- 최종 의사결정 지표 = **YDS 총점** (단일 숫자)
- **패닉매수는 보너스 구간** — 자주 오지 않으며, **최종 목표가 아님**

### 3.2 사이클 흐름

```
🔵 과열 ──→ 🟢 중립 ──→ 🟡 관심 ──→ 🟠 분할매수 ──→ 🔴 패닉매수
  리스크      관찰        1차 기회      핵심 매집        보너스
  관리        준비        · 탐색        · 비중 확대      · 극단 공포
                                                    (드묾)
```

## 4. 구간별 역할 (V1 초안)

| 구간 | 점수 | 역할 | 투자 포지션 |
|------|------|------|-------------|
| 🔵 **과열** | 0–19 | 리스크 관리 · 현금 확보 준비 · 과열 신호 점검 | 추격 금지 · 비중 축소 |
| 🟢 **중립** | 20–39 | 관찰 및 준비 · 시장 모니터링 · 관심 종목 정리 | 탐색 · 보유 유지 |
| 🟡 **관심** | 40–59 | **1차 기회** · 우량 종목 탐색 · 초기 진입 고려 | **준비·선별** (체류 ~44%) |
| 🟠 **분할매수** | 60–79 | **핵심 매집 구간** · 비중 확대 · **주력 구간** | **실전 매집** (체류 ~12%) |
| 🔴 **패닉매수** | 80–100 | 극단적 공포 · 최대 기회 · **보너스** · 자주 오지 않음 | 계획된 공격적 분할 |

## 5. 투자 행동 가이드 (구간별)

### 🔵 과열 (0–19)

- ✓ 기존 수익 보호 · 현금 비중 점검
- △ 신규 추격 진입 금지
- ✗ 레버리지 확대

### 🟢 중립 (20–39)

- ✓ 관심 종목 리스트 정리 · 시장 모니터링
- ✓ 기존 보유 유지
- △ 추격매수 금지

### 🟡 관심 (40–59) — **1차 기회**

- ✓ 우량주 **탐색·선별** · 눌림 목록 작성
- ✓ **초기 분할** 검토 (소량)
- △ 일괄·추격 매수 금지
- *「패닉을 기다리지 말 것」— 관심부터 준비*

### 🟠 분할매수 (60–79) — **핵심**

- ✓ **분할매수 실행** · 비중 **단계적 확대**
- ✓ 우량·현금흐름 중심 · 계획 집행
- △ 한 번에 올인 금지
- *역사적 COVID·2022·관세 — 모두 이 구간*

### 🔴 패닉매수 (80–100) — **보너스**

- ✓ 남은 현금 **우선 투입** · 분할 가속
- △ 레버리지 과다 금지
- *7년 표본 0회 — **있으면 좋은 극단**, 없어도 사이클 완성*

## 7. V1 권장 철학

### 7.1 한 줄

> **「관심에서 준비하고, 분할매수에서 실행한다. 패닉매수는 보너스다.」**

### 7.2 사용자에게 전달할 우선순위

1. **YDS 총점** — 지금 사이클 어디인가 (1순위 UI)
2. **🟡 관심** — 1차 기회 · 준비
3. **🟠 분할매수** — 핵심 매집 · 실행
4. **🔴 패닉매수** — 보너스 · 드묾 · 80+ 기다리지 말 것

### 7.3 엔진 정책 (V1 고정)

- `getFinalScore` · 가중치 · 구간 경계 **유지**
- V1은 **카피·IA·시각 계층**만 사이클 철학에 정렬
- 구간 확대/축소는 **V2 연구** (본 문서 범위 외)

## 8. UI 권장안 (시장분석 페이지)

### 8.1 Before → After

| Before | After |
|--------|-------|
| 패닉매수 = 최종·강조 | **분할매수 = 핵심** (가장 큰 행동 카드) |
| 관심 = 부가 | **관심 = 준비·1차 기회** (총점 바에서 40–59 강조) |
| 80+ 대기 암시 | **「80+는 보너스, 60–79에서 실행」** 명시 |
| 시장 위치 | **YDS 총점** (통일 명칭) |

### 8.2 구체 적용 (엔진 무수정)

- **YDS 총점 스포트라이트**: 40–79 구간 레이블 **「기회 구간」** 보조 텍스트
- **YDS 판단 카드**: 오늘의 행동 — 구간별 위 가이드 매핑
- **YDS 총점 히스토리**: 60–79 밴드 시각 강조 (zone bands)
- **점수 산출 근거**: 「패닉 미달 이유」(HY 가중 등) — 교육용
- **툴팁**: 🔴 패닉매수 = 「보너스 · 7년 표본 0회 · COVID도 75점」

### 8.3 카피 예시

```
🟡 관심구간 — 1차 기회. 우량주 탐색·초기 분할 검토.
🟠 분할매수 — 핵심 매집. 비중 확대·계획 집행.
🔴 패닉매수 — 보너스 구간. 드물게 발생. 80+만이 기회는 아님.
```
